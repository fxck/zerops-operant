Run Operant on a single node: managed Postgres, one Operant container (control plane +
OpenClaw gateway as two processes), and a sandbox Docker host. The simplest, cheapest
way to self-host Operant for a team — ideal for evaluation and steady, low-to-moderate
traffic. There's no redundancy, so a container restart briefly drops chat ingress; step
up to HA Production for a durable HA database.
